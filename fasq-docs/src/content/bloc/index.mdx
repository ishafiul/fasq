# Introduction

Welcome to **Fasq Bloc**!

This adapter brings the power of **Fasq** (caching, deduplication, optimistic updates) to the world of **Discord** (Blocs, Cubits, Streams). It allows you to build data-driven apps with the architecture you already know and love.

## Core Concepts

| Component                                                 | Description                                    |
| :-------------------------------------------------------- | :--------------------------------------------- |
| **[FasqBlocProvider](/docs/bloc/provider)**               | The root widget that sets everything up.       |
| **[QueryCubit](/docs/bloc/query-cubit)**                  | A Cubit that fetches and caches data.          |
| **[MutationCubit](/docs/bloc/mutation-cubit)**            | A Cubit for creating/updating/deleting data.   |
| **[InfiniteQueryCubit](/docs/bloc/infinite-query-cubit)** | Specialized Cubit for infinite lists/feeds.    |
| **[Composition](/docs/bloc/composition)**                 | Mixin to combine multiple queries in one Bloc. |

## Quick Start

1. **[Setup](/docs/bloc/provider)**: Wrap your app with `FasqBlocProvider`.
2. **Create**: Extend `QueryCubit` for your data.
3. **Consume**: Use `BlocBuilder` to show loading, error, or data states.

Check out the **[Examples](/docs/bloc/examples)** to see it in action!
